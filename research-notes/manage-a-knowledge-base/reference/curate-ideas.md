# Curate Ideas

Prune the idea backlog so the good kernels surface and the dead ones stop cluttering it.

## Steps

1. **Read the idea file**: the running list of sparks, kernels, half-thoughts.
2. **Cluster** the kernels by theme.
3. **Mark promotion candidates:** a cluster with several kernels that has also aged past a threshold (it kept coming back, it's not a one-day excitement).
4. **Mark prune candidates:** singletons untouched for a long stretch.
5. **Graduate** the promotions into their own working documents.
6. **Archive** the prunes (don't delete, move them aside with a reason).
7. **Rewrite the idea file** with the survivors, and log before/after counts.

## Non-obvious know-how

- **The promotion gate is recurrence, not enthusiasm.** A cluster graduates only when independent observations show up across multiple sessions. One excited capture, however good it felt, is not a cluster.
- A low input rate is itself a signal: if almost nothing new is arriving, the capture habit upstream needs attention, not the backlog.

## Output

A pruned idea file containing only survivors, new graduated documents for promoted clusters, an archive of pruned singletons, and a before/after counts log.
